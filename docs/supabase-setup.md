# Supabase 接続セットアップ手順

> 2026年4月時点の情報をもとに作成しています。

## 前提

このアプリの「保存する」機能（記事をDBに永続化）はSupabaseを使用します。  
接続しない場合でもRSSフィード閲覧・ローカルの既読管理・後で読む機能は動作します。

---

## 手順

### 1. Supabaseプロジェクトを作成する

ブラウザで **https://database.new** にアクセスするとプロジェクト作成画面に直接遷移します。

または以下の手順で作成します。

1. [https://supabase.com/dashboard](https://supabase.com/dashboard) にサインイン
2. 左上のOrganizationドロップダウンから使用するOrganizationを選択
3. **New project** をクリック
4. 以下を設定して作成

   | 項目 | 推奨値 |
   |---|---|
   | Project name | 任意（例: `rss-news-app`） |
   | Database password | 任意（安全なパスワードを設定） |
   | Region | `Northeast Asia (Tokyo)` |

5. プロジェクトが起動するまで **1〜2分** 待つ

---

### 2. テーブルを作成する

1. プロジェクトのダッシュボードを開く
2. 左サイドバーの **SQL Editor** をクリック
3. 以下のSQLを貼り付けて **Run** をクリック

```sql
create table if not exists public.news (
  id bigint generated by default as identity primary key,
  title text not null,
  link text not null,
  summary text,
  source text not null,
  created_at timestamptz not null default now()
);

create index if not exists news_created_at_idx on public.news (created_at desc);
create index if not exists news_link_idx on public.news (link);
```

> このSQLは `supabase/news.sql` にも保存されています。

---

### 3. APIキーを取得する

1. 左サイドバーの **Settings** → **API Keys** を開く
2. 以下の2つをコピーする

   | 環境変数名 | 取得場所 |
   |---|---|
   | `SUPABASE_URL` | **Project URL** |
   | `SUPABASE_SERVICE_ROLE_KEY` | **Secret key**（`sb_secret_` から始まる形式）|

> **Secret key** が見当たらない場合は **Legacy API Keys** タブの **service_role** キー（JWT形式）を使用しても動作します。  
> ただし公式は新形式（`sb_secret_xxx`）への移行を推奨しています。

> ⚠️ Secret key / service_role キーはサーバー側専用です。ブラウザに公開しないよう `NEXT_PUBLIC_` プレフィックスは付けないでください。

---

### 4. 環境変数を設定する

プロジェクトルートに `.env.local` ファイルを作成し、取得したキーを貼り付けます。

```
SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=sb_secret_xxxxxxxxxxxxxxxxxxxxxxxx
```

> `.env.local` はすでに `.gitignore` に登録済みのため、Gitにコミットされません。

---

### 5. 動作確認する

開発サーバーを起動します。

```bash
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開き、記事の **「保存する」** ボタンが正常に動作すれば接続成功です。

以下のメッセージが表示される場合は、環境変数のコピーミスがないか確認してください。

```
保存機能は現在利用できません。Supabase 環境変数が未設定のため...
```

---

## Vercelにデプロイする場合

`.env.local` はローカル専用のため、Vercelには別途設定が必要です。

### 方法A: Vercel Marketplace経由（推奨）

1. [https://vercel.com/marketplace](https://vercel.com/marketplace) でSupabaseを選択
2. **Install** をクリックしてVercelプロジェクトと連携
3. 環境変数が自動的に注入される

### 方法B: 手動設定

1. Vercelダッシュボードでプロジェクトを開く
2. **Settings** → **Environment Variables** を開く
3. 以下の2つを追加する

   | Name | Value |
   |---|---|
   | `SUPABASE_URL` | SupabaseのProject URL |
   | `SUPABASE_SERVICE_ROLE_KEY` | Secret key（または service_role キー） |

4. **Save** 後、**Deployments** から **Redeploy** を実行
