# RSS News App

複数の RSS をまとめて確認し、気になった記事を保存しながら読むための Next.js アプリです。

本番 URL: `https://rss-news-app.vercel.app`

## 概要

- 複数ソースの RSS をまとめて取得
- ソースごとに初期 3 件表示、必要時だけ `もっと見る`
- 既読 / 未読、後で読む、ソースごとのミュート
- 関連トピックの簡易グルーピング
- 保存済み記事の一覧表示
- 記事タイトルの AI 要約
- 手動での記事保存
- モバイル向けの下部ナビと記事ドロワー

## 使用技術

## セットアップ

```bash
npm install
npm run dev
```

開発サーバーは通常 `http://localhost:3000` で起動します。

## 環境変数

`.env.local` などに以下を設定します。

```env
SUPABASE_URL=...
SUPABASE_SERVICE_ROLE_KEY=...
OPENAI_API_KEY=...
OPENAI_SUMMARY_MODEL=gpt-5-nano
```

### 役割

- `SUPABASE_URL`
  保存済み記事の読み書きに使用
- `SUPABASE_SERVICE_ROLE_KEY`
  `news` テーブルへのサーバー側アクセスに使用
- `OPENAI_API_KEY`
  `/api/summarize` で記事タイトル要約に使用
- `OPENAI_SUMMARY_MODEL`
  任意。未指定時は `gpt-5-nano`

`SUPABASE_URL` または `SUPABASE_SERVICE_ROLE_KEY` が未設定の場合、保存済み記事まわりは利用できません。

## Supabase テーブル

保存機能は `news` テーブルを前提にしています。最低限、以下のカラムが必要です。

```sql
create table public.news (
  id bigint generated by default as identity primary key,
  title text not null,
  link text not null unique,
  summary text default '',
  source text not null,
  created_at timestamptz not null default now()
);
```

## API

- `/api/rss`
  登録済み RSS を取得
- `/api/news`
  保存済み記事一覧を取得
- `/api/news/save`
  記事を保存
- `/api/news/search`
  保存済み記事タイトル検索
- `/api/summarize`
  記事タイトルの AI 要約
